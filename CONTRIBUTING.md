# Contributing to LeadGenius

Thank you for your interest in contributing to LeadGenius! 🎉

## 🚀 Quick Start

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/LeadGenius.git`
3. Install dependencies: `npm install`
4. Create a branch: `git checkout -b feature/your-feature`
5. Make your changes
6. Push and create a Pull Request

## 📋 Guidelines

### Code Style

- Use TypeScript for all new files
- Follow the existing code structure
- Add comments for complex logic
- Write meaningful commit messages

### Commit Messages

```
type: short description

# Examples:
feat: add multi-language support
fix: resolve email sending issue
docs: update README
refactor: clean up API routes
```

### Pull Requests

- Describe what your PR does
- Link any related issues
- Test your changes before submitting
- Keep PRs focused on one feature/fix

## 🐛 Reporting Bugs

1. Check existing issues first
2. Use the bug report template
3. Include:
   - Steps to reproduce
   - Expected behavior
   - Actual behavior
   - Screenshots if applicable

## 💡 Requesting Features

1. Check existing issues first
2. Use the feature request template
3. Describe:
   - What the feature does
   - Why it would be useful
   - How it should work

## 🧪 Testing

```bash
npm run test        # Run tests
npm run lint        # Check code style
npm run typecheck   # Type check
```

## 📝 Documentation

- Update README.md if needed
- Add JSDoc comments to new functions
- Update .env.example for new env vars

Every contribution helps make LeadGenius better!
